Caracterización de Requisitos No Funcionales

Aplicación seleccionada: Instagram

Introducción

Los requisitos no funcionales (RNF) definen atributos de calidad que permiten que una aplicación funcione correctamente más allá de sus funcionalidades visibles. En plataformas masivas como Instagram, estos requisitos impactan directamente la experiencia del usuario, la arquitectura del sistema y la infraestructura tecnológica.

1. Historias

Definición

Las historias son publicaciones temporales de fotos o videos que desaparecen después de 24 horas.

Disponibilidad

99.9%

Fiabilidad del servicio de historias.

Publicación

< 5s

Tiempo máximo de carga.

RNF 1: Fiabilidad – Disponibilidad

  • Fuente: Usuario de Instagram
  • Estímulo: Intenta ver una historia
  • Artefacto: Servicio de historias + almacenamiento multimedia
  • Entorno: Hora pico con alta concurrencia
  • Respuesta: El sistema carga la historia correctamente
  • Medida: Disponibilidad ≥ 99.9% mensual

Trade-off

Mayor disponibilidad requiere más servidores redundantes y mayor costo.

Implicaciones arquitectónicas

  • Infraestructura distribuida
  • Balanceo de carga
  • Replicación de contenido

Tácticas

  • Redundancia activa
  • Failover automático

RNF 2: Rendimiento – Comportamiento temporal

  • Fuente: Usuario
  • Estímulo: Publica una historia
  • Artefacto: Servicio de carga multimedia
  • Entorno: Operación normal
  • Respuesta: Historia publicada
  • Medida: Tiempo total < 5 segundos

Trade-off

Mayor velocidad puede reducir calidad del archivo.

Implicaciones

  • Procesamiento paralelo
  • CDN
  • Caché

Tácticas

  • Cola asíncrona
  • CDN cercana al usuario

2. Filtros de cámara

Definición

Son efectos visuales de realidad aumentada aplicados en tiempo real sobre la imagen capturada por la cámara.

Latencia

< 100ms

Respuesta en tiempo real.

Seguridad

0 fugas

Protección de datos biométricos.

RNF 1: Rendimiento

  • Fuente: Usuario
  • Estímulo: Cambia filtro
  • Artefacto: Motor gráfico / cámara
  • Entorno: Dispositivo gama media
  • Respuesta: Filtro aplicado en tiempo real
  • Medida: Latencia < 100 ms

Trade-off

Mayor calidad gráfica aumenta consumo de batería.

Implicaciones

  • Uso de GPU
  • Procesamiento local

Tácticas

  • Renderizado por hardware
  • Precarga de filtros

RNF 2: Seguridad – Confidencialidad

  • Fuente: Usuario
  • Estímulo: Activa filtro facial
  • Artefacto: Datos capturados por cámara
  • Entorno: Sesión normal
  • Respuesta: Datos protegidos
  • Medida: 0 fugas / cifrado habilitado

Trade-off

Mayor seguridad puede aumentar consumo de recursos.

Implicaciones

  • Cifrado
  • Políticas de privacidad

Tácticas

  • Cifrado en tránsito y reposo
  • Procesamiento local

3. Feed personalizado

Definición

Pantalla principal donde el contenido es ordenado por algoritmos según gustos e interacciones previas.

Disponibilidad

99.95%

Alta disponibilidad global.

Respuesta

< 2s

Entrega rápida del feed.

RNF 1: Fiabilidad – Disponibilidad

  • Fuente: Usuario
  • Estímulo: Abre la aplicación
  • Artefacto: Servicio feed
  • Entorno: Alta demanda global
  • Respuesta: Feed cargado correctamente
  • Medida: Disponibilidad ≥ 99.95%

Trade-off

Mayor disponibilidad implica más complejidad y costo.

Implicaciones

  • Microservicios resilientes
  • Replicación regional

Tácticas

  • Autoescalado horizontal
  • Circuit breaker

RNF 2: Rendimiento – Capacidad

  • Fuente: Millones de usuarios
  • Estímulo: Solicitan feed al mismo tiempo
  • Artefacto: Motor de recomendación
  • Entorno: Hora pico
  • Respuesta: Feed entregado sin lentitud
  • Medida: < 2 segundos

Trade-off

Mayor personalización consume más procesamiento y datos.

Implicaciones

  • Sistemas distribuidos
  • Bases escalables
  • Machine Learning

Tácticas

  • Caché de recomendaciones
  • Procesamiento distribuido

Conclusión

El éxito de Instagram depende de que sus funcionalidades sean rápidas, seguras, disponibles y escalables.
